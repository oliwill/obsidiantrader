# /think - 分析前思考

## 何时使用
开始任何新的股票分析前

## 目标
- 明确分析目标和边界
- 识别潜在盲点
- 选择合适的数据源和分析方法
- 确保分析输出符合 10 组件新框架

## 检查清单

### 1. 理解需求
- [ ] 用户想要基本面、技术面还是综合分析？
- [ ] 时间范围是否明确？
- [ ] 这只股票属于什么行业/市场？

### 2. 数据准备
- [ ] 股票代码是否有效？
- [ ] 数据是否足够？（新股、停牌等情况）
- [ ] 需要补充哪些外部数据？

### 3. 分析方法选择
- [ ] 根据股票类型选择模型（成长股、价值股、周期股）
- [ ] 是否需要特殊处理？（ST股、次新股、外资股）
- [ ] 有哪些已知的风险因素？

### 4. 输出规划
- [ ] 用户需要什么样的结论？
- [ ] 需要哪些可视化图表？
- [ ] 风险提醒是否充分？

## 五维打分体系（第 4.5 步）

### 维度 1：行业 / TAM（满分 10 分）
子维度（每项 ★-★★★★★）：
- 赛道方向：是否处于结构性增长赛道？
- TAM 规模：市场天花板是否足够大？
- 资金流向：板块是否有持续资金流入？

打分参考：
- 8-10 分：AI CAPEX 超级周期级别，多因素叠加
- 5-7 分：稳健增长赛道，但非超级周期
- 0-4 分：成熟/衰退行业

### 维度 2：护城河（满分 10 分）
子维度：
- 技术/IP：专利壁垒、独特技术积累
- 客户锁定：切换成本、平台绑定程度
- 规模：成本结构、议价能力

打分参考：
- 8-10 分：多维度护城河，竞争对手短期无法复制
- 5-7 分：有一定护城河但有窗口期
- 0-4 分：护城河弱或正在被侵蚀

### 维度 3：增长速度与质量（满分 10 分）
子维度：
- 历史增长：过去 2 年实际增速
- 前瞻增长：未来 2 年预期增速
- 增长质量：毛利率趋势、FCF、经常性收入占比

打分参考：
- 8-10 分：YoY > 30% + 毛利率扩张 + FCF 正
- 5-7 分：YoY 10-30% 或增速放缓但质量改善
- 0-4 分：YoY < 5% 或负增长（转型期需特殊说明）

### 维度 4：估值（满分 10 分）
子维度：
- PSG 水平（P/S ÷ 增速%）
- 同行对比溢价
- 安全边际

打分参考：
- 8-10 分：PSG < 0.5，显著低于同行，安全边际充足
- 5-7 分：PSG 0.5-1.5，合理偏高
- 0-4 分：PSG > 1.5，无安全边际

### 维度 5：团队 / 治理（满分 10 分）
子维度：
- CEO 背景：行业经验、过往业绩
- 董事会质量：独董背景、治理结构
- 内部人信号：过去 6 个月买入/卖出比
- Skin in the game：管理层持股比例、SBC 占收入比

打分参考：
- 8-10 分：行业老兵 CEO + 内部人净买入 + 低 SBC
- 5-7 分：CEO 合格但内部人信号中性
- 0-4 分：内部人大量减持或 SBC 占收入 > 50%

### 综合判定阈值
- 加权平均 ≥ 7.5 → 高信心建仓（最大仓位 5%）
- 6.0-7.5 → 标准建仓（最大仓位 3%）
- 4.5-6.0 → 观察 / 小仓位试探（最大仓位 1-2%）
- < 4.5 → Pass

### 输出格式
```
五维打分:
- 行业/TAM: X/10（赛道方向 ★★★★★, TAM规模 ★★★★☆, 资金流向 ★★★★☆）
- 护城河: X/10（技术IP ★★★★☆, 客户锁定 ★★★☆☆, 规模 ★☆☆☆☆）
- 增长: X/10（历史增长 ★☆☆☆☆, 前瞻增长 ★★★☆☆, 增长质量 ★★☆☆☆）
- 估值: X/10（PSG=X, 同行溢价=高, 安全边际=无）
- 团队: X/10（CEO ★★★★★, 董事会 ★★★★☆, 内部人信号 ★☆☆☆☆, SBC ★★☆☆☆）
综合: X/10 → [判定结果]
```

## 不对称原型分类（第 4.7 步）

在完成五维打分后，判断这只股票属于哪种赚钱模式：

### 四种原型

| 原型 | 特征 | 典型PSG | 持仓周期 | 适合策略 |
|------|------|---------|---------|---------|
| 高确定性增长 | 稳定YoY>20%+合理估值+多支柱 | < 1.0 | 6-12个月 | 分批建仓，长期持有 |
| 催化剂驱动 | 等特定事件兑现（财报/合作/审批） | 1.0-3.0 | 1-3个月 | 事件前建仓，兑现后评估 |
| 均值回归 | 被错杀，基本面健康但价格偏离 | < 0.5 | 3-6个月 | 左侧建仓，等市场修正 |
| 困境反转 | 管理层变革/业务重构，高风险高回报 | 任意 | 6-18个月 | 极小仓位，分批加码 |

### 分类检查
- [ ] 这只股票属于哪种原型？（可多选，但必须标注主次）
- [ ] 当前价位是否提供了非对称赔率？（上行空间 > 2× 下行风险）
- [ ] 如果是催化剂驱动型：催化剂日期是否明确？估值是否已经透支了催化兑现？

### 关键判断
⚠️ **如果一只股票符合催化剂驱动型，但当前估值已经透支了催化兑现，非对称赔率变成负向 → 这是框架要规避的情境。**

## 估值锚点准备（第 5 步）

在开始写分析前，先完成以下计算：

#### 必填数据
- [ ] 当前市值（从 `stock_info.market_cap` 获取）
- [ ] TTM 年化营收（最近4季度之和，或直接从 `fundamentals` 获取）
- [ ] **当前 P/S = 市值 ÷ 年化营收**（若亏损公司 PE 无意义，P/S 是主锚）
- [ ] 预期收入增速%（取 `revenue_growth` 或分析师共识）
- [ ] **PSG = P/S ÷ 增速%**

#### PSG 判断标准（来自实战验证）
- **PSG < 1** → 合理估值，有安全边际
- **PSG 1–2** → 偏高，谨慎
- **PSG > 2** → 极度高估，无安全边际
- **PSG > 10** → 完全脱离现实（如 POET 的 PSG 199）

#### 同行对比
- [ ] 找 3–5 家同行的 P/S（web search "[sector] peer P/S 2026"）
- [ ] 至少包含：龙头公司（如 NVDA）、纯玩同行（如 LITE/COHR）、传统同行（如 AAOI）
- [ ] 制作对比表格，说明当前公司的相对位置

#### 结论模板
```
估值锚点：当前 P/S = X，同行均值 Y，PSG = Z（[合理/偏高/极度高估]）
安全边际判断：[充足 / 偏紧 / 无安全边际]
关键句：即使 [年份] 营收翻 [N] 倍到 $X，按同行 [Y]x P/S 估值，市值仅 $Z，较现在还要跌 -N%
```

### 估值透支压力测试

在完成基本估值锚点后，做一次极端乐观压力测试：

**问题：即使最好的情况发生，当前估值能不能撑住？**

计算步骤：
1. 假设公司未来 N 年每年营收翻倍（或分析师最乐观预期）
2. 计算该情景下的远期 P/S
3. 与同行均值 P/S 对比

```
压力测试：假设 [年份] 营收达到 $X（每年翻倍）
远期 P/S = 当前市值 ÷ $X = Y
同行均值 P/S = Z
结论：[勉强合格 / 仍然高估 / 有安全边际]
→ 当前估值已透支了 [1. 每年大超预期的翻倍财报、2. 一直到 YYYY年底的营收]
```

如果压力测试结果是"仍然高估" → 当前价格不值得追入，即使在 Bull Case 下也缺乏安全边际。

## 三情景框架准备（第 6 步）

在写 Bull/Base/Bear 前，先确认：

### 核心驱动力识别
- [ ] **核心驱动力是什么**（一条或两条，整个 thesis 的支柱）
  - 格式："公司是 [领域] 的 [定位]，核心驱动力是 [具体事件/趋势]"
  - 示例："POET 是光子集成半导体公司，核心驱动力是 Marvell + NVIDIA 光连接订单"

### 最大单点风险识别
- [ ] **最大的单点风险是什么**（这条风险如果发生，thesis 是否直接崩塌）
  - 格式："如果 [风险事件]，[影响量化]"
  - 示例："如果 Marvell 订单 < $3M，2026 年收入预期从 $10M 下调到 $3M → 估值压缩 60–70%"

### 三情景设计
- [ ] **Bull 假设**：核心驱动力全部兑现 + 无重大风险触发
  - 核心假设：3–5 条（如订单金额、合作落地、增长超预期）
  - 估值方法：明确说"用 [指标] × [倍数] = [市值]"
  - 概率：20–30%

- [ ] **Base 假设**：核心驱动力部分兑现（如延迟6–12个月）
  - 核心假设：3–5 条（比 Bull 保守）
  - 估值方法：向同行均值回归
  - 概率：45–55%

- [ ] **Bear 假设**：核心风险触发，收入预期被大幅下调
  - 核心假设：3–5 条（风险事件列表）
  - 估值方法：P/S 崩至行业底部（如 15x）
  - 概率：20–30%

- [ ] **概率之和 = 100%**

### 概率加权目标价计算
```
加权价 = Bull价 × Bull概率 + Base价 × Base概率 + Bear价 × Bear概率
隐含回报 = (加权价 - 当前价) / 当前价 × 100%
```

### 结论模板
```
当前价 $X vs 加权目标价 $Y → 隐含 12 个月回报 ±Z%
一句话判断：[满足/不满足] 建仓门槛（理由）
```

## 操作格网设计（第 7 步）

### 价位档位设计原则
至少 5 个档位：

| 档位 | 价位区间 | 动作 | 仓位 | 触发条件 |
|------|---------|------|------|----------|
| 当前价 | $X | 🔴 追/不追 | 0% | PSG/IV 判断 |
| 回调位1 | $Y | 🟡 小仓位试探 | 0.5% | 催化剂部分兑现 |
| 核心建仓 | $Z | 🟢 建仓第一批 | 1% | 催化剂全面兑现 + 财报验证 |
| 深度加仓 | $W | 🟢 加仓第二批 | 1.5% → 2.5% | Base/Bear 概率再平衡 |
| 极端值 | $V | 🟢 重仓 | 2.5% → 3% | Bear 触发但市场错杀 |

### 档位定价逻辑
- **当前价**：直接用 PSG 判断，PSG > 2 通常不追
- **回调位1**：MA20/MA50 技术支撑 + 情绪降温
- **核心建仓**：对应 PSG < 1 的合理估值
- **深度加仓**：对应 Bear Case 底部估值
- **极端值**：对应行业危机时的错杀价

### 最大仓位上限
- 高风险标的（如 PSG > 10）：单仓 ≤ 2%
- 中等风险（PSG 2–10）：单仓 ≤ 3%
- 低风险（PSG < 1）：单仓 ≤ 5%

如果 `data/earnings.py` 返回的下一次财报日期距今 < 30 天：

| 情景 | 概率 | 关键条件 | 股价反应 |
|------|------|---------|---------|
| Hyper-beat | X% | Revenue > $X + 下季指引 > $Y + [核心催化剂] | ±X% → $Y |
| In-line | X% | Revenue $X-Y + 下季指引持平 | ±X% → $Y |

隐含一周回报：±Y%

**如果加权预期股价 < 当前价 → 财报前不是好的入场时机（sell-the-news 风险）**

## 市场结构信息收集（第 8 步）

在 web search 阶段同步查找：

### 必填项
- [ ] **Short Float %**（空头占流通股比例）
  - 搜索："[TICKER] short interest"
  - 判断：< 5% 正常，5–10% 关注，> 10% 高关注（有 squeeze 风险）
  - 如果快速上升（如 6% → 8.6%），说明空头建仓中

- [ ] **期权 IV**（Implied Volatility）与历史均值对比
  - 搜索："[TICKER] implied volatility options"
  - 判断：< 50% 正常，50–100% 偏高，> 100% 极端
  - 示例：POET 的 IV 148–157%

- [ ] **内部人增减持**（过去6个月）
  - 搜索："[TICKER] insider trading buying selling 2026"
  - 收集：买入次数、卖出次数、累计金额（净买入 or 净卖出）
  - 特别关注：CEO/CFO 个人交易（排除 10b5-1 计划性卖出）

  **内部人信号评分**：
  | 信号 | 评分 |
  |------|------|
  | 净买入 > $1M + CEO/CFO 个人买入 | ★★★★★ 极正面 |
  | 净买入 > $500K | ★★★★☆ 正面 |
  | 买入 ≈ 卖出（中性） | ★★★☆☆ |
  | 净卖出 > $5M + 0 次买入 | ★☆☆☆☆ 极负面 |
  | 净卖出 > $10M + CFO 在窗口期卖出 | ☆☆☆☆☆ 危险信号 |

  **同行横向对比**（如果可能）：
  将当前股票的内部人信号与同行业 2-3 家公司对比，判断是否为行业普遍现象（如年底计划性卖出）还是个股特有风险。

### 可选项
- [ ] 机构持股变化（最近一个季度 13F 变化）
- [ ] 近期涨幅中，是否有明显的空头回补（short squeeze）成分

### 拆解格式
```
近期涨幅拆解：过去 [N] 日从 $X → $Y（±Z%）
- 基本面新信息贡献：[催化剂名称] — 真实催化剂，但 [细节未披露/已量化]
- 技术性因素贡献：空头回补（Short Float X%）+ 期权 Gamma + 情绪 FOMO — 估算占 [X]%
判断：当前价格中，情绪溢价占比过高/适中，风吹草动会快速回吐
```

## 风险量化模板（第 9 步）

### 必填 3–5 条风险，每条格式：
```
[风险名称] → 如果发生：[收入影响 ±X%，估值影响 ±Y%，概率判断（高/中/低）]
```

### 风险分类建议
1. **核心驱动力风险**（最重要）：如"Marvell 订单金额不及预期"
2. **合作落地风险**：如"NVIDIA 合作未落地"
3. **估值回归风险**：如"P/S 向同行均值回归"
4. **历史规律风险**：如"过去炒作浪潮后营收未增长"
5. **内部人信号风险**：如"内部人持续减持"
6. **SBC/稀释风险**：如"SBC 占收入比例 > 50%"、"过去一年股本增加 > 10%"

### 头号风险标识
在 3–5 条风险中，明确标注：
```
**头号风险**：[风险名称] — 这条风险如果发生，整个 thesis 的支柱被抽走。
```

### SBC / 股权稀释专项检查（适用于小市值成长股）

- [ ] SBC 占收入比例（web search "[TICKER] stock based compensation revenue"）
  - < 10% → 正常
  - 10-30% → 偏高
  - > 30% → 警告
  - \> 50% → 极端稀释

- [ ] 过去 12 个月股本稀释率
  - < 3% → 正常
  - 3-10% → 偏高
  - \> 10% → 严重稀释

- [ ] 是否有可能再融资？（现金储备 ÷ 季度烧钱率 < 8 个季度 → 可能再融资）

如果 SBC 占收入 > 50% 或年稀释 > 10%，在风险量化中必须作为独立风险条目，并标注：
"**SBC 风险**：SBC 占收入 X%，意味着公司每赚 $1 就要发 $Y 股票给员工 — 真实股东回报被持续稀释。"

## 警戒线/加仓信号模板（第 10 步）

### 🔴 警戒线（3–6条）

每条必须是**可观测的布尔条件**：
```
**如果 [具体事件 A]** → [动作]，[原因]
```

错误示范：
- ❌ "如果市场好转"（模糊）
- ❌ "如果公司业绩好"（不可量化）

正确示范：
- ✅ "如果 Marvell 披露订单金额 < $3M"（可观测）
- ✅ "如果 Q1 财报营收 < $1.5M"（可量化）
- ✅ "如果 P/S 在财报后仍 > 500x"（可计算）
- ✅ "如果 NVIDIA 官方否认与 POET 的采购关系"（可验证）

### 🟢 加仓信号（3–6条）

同样格式：
```
**如果 [具体事件 B]** → 可在 [价位] 加仓至 [仓位]，[原因]
```
