# /check - 分析结果检查

## 何时使用
完成分析后，输出结果前

## 目标
- 验证分析逻辑的合理性
- 检查数据异常
- 确保所有 10 个必要组件存在且可执行

## 检查清单

### 1. 数据验证
- [ ] 价格数据是否在合理范围？
- [ ] 财务指标是否与行业平均水平偏离过大？
- [ ] 时间序列是否连续？

### 2. 逻辑检查
- [ ] 分析结论是否与数据一致？
- [ ] 多空观点是否平衡？
- [ ] 是否存在幸存者偏差？

### 3. 风险提示
- [ ] 是否说明了分析的局限性？
- [ ] 是否提及了重大不确定性？
- [ ] 是否强调了不构成投资建议？

### 4. 可解释性
- [ ] 关键结论是否有数据支撑？
- [ ] 技术术语是否解释清楚？
- [ ] 评分依据是否透明？

### 5. Obsidian Front Matter 完整性
- [ ] 报告开头是否有 YAML front matter（用 `---` 包围）？
- [ ] `title` 格式是否正确（"{TICKER} {评分}/100 — {核心观点简述}"）？
- [ ] `description` 是否是一句话精华（含 PSG、目标价、核心操作）？
- [ ] `tags` 是否包含 `stock-analysis`？
- [ ] `tags` 是否包含行业标签（如 `semiconductors`）？
- [ ] `tags` 是否包含 `12m-target-{价格}`？
- [ ] `tags` 是否包含 `psg-{值}`？
- [ ] `score`、`psg`、`target_price`、`current_price` 字段是否填入？
- [ ] `published`、`created` 日期是否正确？

---

## 10 组件完整性检查（必查）

### 6. 估值锚点完整性
- [ ] 是否计算了当前 TTM P/S？
- [ ] 是否有 3 家以上同行 P/S 对比（含龙头、纯玩、传统）？
- [ ] 是否计算了 PSG 并给出判断（合理/偏高/极度高估）？
- [ ] 估值结论是否与操作信号一致？（如"极度高估"却给出"BUY"需要特别说明）
- [ ] 是否有"即使翻倍也难以支撑当前估值"的量化论述？

### 6. 三情景目标价完整性
- [ ] 是否有 Bull / Base / Bear 三个情景？
- [ ] 三情景概率之和是否等于 100%？
- [ ] 每个情景是否有 3–5 条核心假设？
- [ ] 是否计算了概率加权目标价？
- [ ] 加权目标价 vs 当前价的隐含回报是否明确写出？
- [ ] 是否有"一句话判断：满足/不满足建仓门槛"？

### 7. 风险量化完整性
- [ ] 是否列出至少 3 个主要风险？
- [ ] 每条风险是否说明了"如果发生，对收入/估值的影响是 X%"？
- [ ] 是否标注了头号风险（最大单点风险）？
- [ ] 头号风险是否与 thesis 的核心支柱对应（即：如果最大风险发生，整个 thesis 是否崩塌）？

### 8. 操作格网完整性
- [ ] 是否明确说明当前价"追/不追"？
- [ ] 是否有至少 2 个价位区间的具体买入条件？
- [ ] 是否说明了最大仓位上限？
- [ ] 每个价位的"触发条件"是否可观测（避免"市场好转"等模糊表述）？
- [ ] 档位设计是否合理（当前价 → 回调位1 → 核心建仓 → 深度加仓 → 极端值）？

### 9. 警戒线/加仓信号完整性
- [ ] 是否有至少 3 条警戒线（具体、可观测的布尔条件）？
- [ ] 是否有至少 3 条加仓信号（具体、可观测的布尔条件）？
- [ ] 警戒线是否覆盖了 thesis 最大的单点风险？
- [ ] 每条条件是否可验证（避免模糊表达）？

### 10. 市场结构分析完整性
- [ ] 是否提到空头比例（Short Float %）？
- [ ] 是否对近期涨幅做了基本面 vs 技术性因素的拆解？
- [ ] 是否提到内部人增减持情况？
- [ ] 是否提到期权 IV 与历史均值对比？

## 数据准确性检查（常见错误）

### 估值计算错误
- [ ] P/S = 市值 ÷ 营收（而非 市值 ÷ 净利润）
- [ ] PSG = P/S ÷ 增速%（而非 P/S × 增速%）
- [ ] 增速% 用的是预期增速（而非历史增速）

### 概率计算错误
- [ ] Bull + Base + Bear = 100%
- [ ] 概率分配合理（避免 Bull 80% 这种极端）
- [ ] 加权价计算正确（Bull×P + Base×P + Bear×P）

### 同行对比错误
- [ ] 至少包含 3 家同行
- [ ] 同行选择合理（同一行业、可比市值）
- [ ] P/S 数据来源可靠（web search 查询）

---

## 逻辑一致性检查

### 估值 vs 操作信号
- [ ] 如果 PSG > 10（极度高估），是否给出"不追"？
- [ ] 如果 PSG < 1（合理估值），是否给出"可建仓"？
- [ ] 估值判断与操作格网一致（避免矛盾）

### 三情景 vs 风险
- [ ] Bear 假设是否与最大风险对应？
- [ ] Bull 假设是否与核心驱动力对应？
- [ ] Base 假设是否是"中间状态"（非极端）？

### 风险 vs 警戒线
- [ ] 警戒线是否覆盖了所有主要风险？
- [ ] 头号风险是否有对应的警戒线？

## 自检输出

```
✓ 数据验证通过
✓ Front Matter: title/description/tags/score/psg/target_price 已填入
✓ 估值锚点: P/S=X，同行均值 Y，PSG=Z（[合理/偏高/极度高估]）
✓ 三情景: Bull $X (P%) / Base $Y (P%) / Bear $Z (P%) → 加权 $W，隐含回报 ±R%
✓ 风险量化: N条，每条含影响量化，头号风险已标注
✓ 操作格网: N档价位，含最大仓位说明
✓ 警戒线/加仓信号: 各N条布尔条件
✓ 市场结构: 空头比例、IV、内部人信号已覆盖
⚠ 需要关注: xxx
✗ 缺少: xxx（需要补充后再输出）
```

## 常见问题排查

| 问题 | 可能原因 | 解决 |
|------|----------|------|
| PSG 计算结果 > 1000 | 增速% 用错（用了 0.x 而非 %）| 确认增速是 10.75 而非 0.1075 |
| 加权目标价 > Bull 目标价 | 概率分配错误 | 检查 Bull + Base + Bear 是否 = 100% |
| 同行 P/S 对比缺失 | web search 未执行 | 补充 web search |
| 操作格网只有 1 档 | 未设计回调位 | 至少要有 5 档（当前价 + 4 个回调位）|
| 警戒线模糊 | 条件不可观测 | 改为具体数字或事件（如"Q1 营收 < $X"）|

## 输出前最终确认

在把分析发给用户前，问自己三个问题：

1. **如果我是用户，我能根据这份分析立刻知道买不买吗？**
   - 如果不能，操作格网需要更明确

2. **如果我的核心假设错了，我清楚会亏多少吗？**
   - 如果不清楚，Bear Case 需要更具体

3. **如果三个月后股价下跌 50%，我能找到是哪个假设失效了吗？**
   - 如果不能，警戒线需要更详细
