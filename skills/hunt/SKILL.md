# /hunt - 问题排查

## 何时使用
分析过程出现异常、结果不合理或收到负面反馈时

## 目标
系统性地定位问题根源

## 排查流程

### Step 1: 复现问题
- [ ] 使用相同参数重新运行
- [ ] 记录完整的错误信息
- [ ] 确认是偶发还是必现

### Step 2: 数据层排查
```python
# 检查数据获取
- 股票代码是否正确？
- API 返回是否成功？
- 数据时间范围是否正确？
- 是否有缺失值/异常值？
```

### Step 3: 分析层排查
```python
# 检查计算逻辑
- 指标计算是否正确？
- 参数设置是否合理？
- 边界条件处理是否完整？
```

### Step 4: 输出层排查
```python
# 检查结果呈现
- 结论是否与数据匹配？
- 图表是否正确渲染？
- 文字描述是否准确？
```

### Step 5: 根因分析
- [ ] 是数据问题、代码问题还是模型问题？
- [ ] 是否影响其他股票分析？
- [ ] 如何防止类似问题？
